# Workflow: Create Video Ad

## Objective
Produce a story-driven short-form video ad (15s/30s/60s) for a client exhibition that drives ticket sales.

## When to Use
- New client onboarded with exhibition photos and brand brief
- New campaign or exhibition added to existing client
- Refreshing content for seasonal push

## Required Inputs
| Input | Source | Notes |
|-------|--------|-------|
| Exhibition photos (5+ hero shots) | Scraped or client-provided | Min 800px wide, JPG/PNG. If quality is low, generate in Midjourney |
| Video scripts | `03_content/{exhibition}/video_scripts_*.json` | Generated by content_creator agent |
| Brand colors | `content_utils.CAMPAIGN_COLORS` | Accent color per exhibition |
| Story sequence | `tools/generate_motion_clips.py` → `STORY_SEQUENCES` | Must define narrative arc |

## Pipeline Steps

### Step 1: Curate Images
**Manual step.** Review available photos and select 3-5 hero shots that tell a story:
- HOOK image — dramatic, jaw-dropping, stops the scroll
- DISCOVERY image — detail or reveal that draws you in
- WONDER image — the core experience, what makes it special
- CONNECTION image — emotional or social moment
- DESIRE image — leaves them wanting more

Add selections to `STORY_SEQUENCES` in `generate_motion_clips.py` with tailored prompts.

### Step 2: Generate Motion Clips
```bash
# Dry run first (free)
python tools/generate_motion_clips.py --exhibition {name} --dry-run

# Generate (costs ~$1.75 per exhibition at Kling 2.6 Pro)
python tools/generate_motion_clips.py --exhibition {name} --max-clips 5
```

**Output:** `.tmp/motion_clips/{exhibition}/01_hook_*.mp4` through `05_desire_*.mp4`

**Quality check:** Open each clip. Verify:
- Smooth motion, no artifacts or distortion
- Matches the creative intent (subject animating, not just camera movement)
- Resolution 720p+ at 24fps

**If quality is poor:** Re-generate with different prompt, or generate a Midjourney image first.

### Step 3: Compose Video Ad
```bash
python tools/compose_video_ad.py --exhibition {name} --duration 15
```

**Output:** `04_deliverables/video_ads_v2/{exhibition}/{exhibition}_15s_*.mp4`

**Quality check:** Play the full video. Verify:
- Story flows naturally between scenes
- Text overlays are readable (not cut off, proper contrast)
- Transitions are smooth (no jarring cuts)
- CTA scene has correct branding and text
- Total duration matches target (within 0.5s)

### Step 4: Full Pipeline (Optional)
```bash
# Run everything end-to-end
python tools/content_engine.py --exhibition {name} --video-durations 15,30

# Or all exhibitions
python tools/content_engine.py --exhibition all
```

## Edge Cases

### API Rate Limits
- fal.ai has generous limits but can queue during peak hours
- Built-in retry logic (3 attempts, 10s delay between)
- If persistent failures, wait 5 minutes and retry

### Low-Resolution Source Images
- If scraped images are <800px wide, quality will suffer
- Solution: Generate hero shots in Midjourney using exhibition-specific prompts
- Store generated images in `media/proposal_ready/{exhibition}/` with `mj_` prefix

### Missing Video Scripts
- If no `video_scripts_*.json` exists, compose_video_ad.py will use generic text
- Better: Run content_creator agent first to generate proper scripts

### Clip Validation Failure
- MoviePy validates each downloaded clip (duration, resolution, codec)
- If validation fails, the clip file is kept for manual review
- Re-generate with a tweaked prompt if the AI output is unusable

## Lessons Learned
- **Source image quality is everything.** A 50KB JPEG produces garbage. Use 300KB+ hero shots.
- **Kling 2.6 Pro >> 2.5 Turbo** for quality. The cost difference is negligible.
- **Alphabetical image selection is lazy.** Every image must serve a narrative purpose.
- **The Playwright slideshow approach is dead.** Don't go back to HTML→screenshot→MP4.
- **Story arc drives conversions.** Hook → Discovery → Wonder → Connection → Desire.
